Storage Repositories

Files are split into chunks and distributed across these repositories. Link extra GitHub accounts for more storage or automatic backup copies.

Linked GitHub Accounts

Copy a one-time link and open it in a private/incognito window (no vault sign-in needed). Authorize with the GitHub account you want to link. Links are generated fresh each time you open this panel.

Backup accounts fork your storage repos (fast full copy). Private repos must be public or add the backup user as a collaborator on each storage repo. Ongoing changes sync via GitHub fork upstream merge.

Uses the same GitHub OAuth app as sign-in. Callback URL must be (your app URL)/auth/github/callback (create app · )

Vault Organization

Create a free org on GitHub, then set it up here. New storage repos will be created under that org instead of your personal account.

Create org on GitHub

MySQL Database

Migrate all local SQLite data to MySQL. Set MYSQL_HOST, MYSQL_USER, MYSQL_PASSWORD, and MYSQL_DATABASE in .env before migrating.

After migration, set USE_MYSQL=true in .env to use MySQL as the primary database.

New storage repos are public by default so share links fetch encrypted chunks directly from GitHub (server optional). Data stays encrypted — only ciphertext is visible. Use the button above for repos created before this change.

Large file upload

        
      

      

      

        
        
        
        
        
Larger chunks = fewer API calls. GitHub allows max 100 MB per file — encrypted chunks are capped at 95 MB.

        
        
HLS uses stream copy (~same size as the original) in addition to encrypted chunks. Total storage reserved is roughly 2× the file size.

Local upload (LAN)

        
      

      

        
When you open the vault via a domain or tunnel, uploads still go through the public path. Save your server’s local IPv4 so you get a one-click link to the fast LAN address on this network.

        
        
        
Private address only (192.168.x.x, 10.x.x.x, 172.16–31.x.x). Find it on the server with ipconfig (Windows) or ip addr (Linux).

        

        

      

      

        
        
        
        
      

    

  


  
  

    

      

        
Cache disk settings

        
      

      

        
Local disk cache stores decrypted files and streaming segments for faster playback and repeat downloads.

        
        
        
        
        

        
Stale entries are cleaned automatically during server maintenance. Lowering the size limit evicts oldest entries immediately.
